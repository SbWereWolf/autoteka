# 1. Выбирай русский язык

- Пиши на русском
- Если создаешь любой проектный текстовый артефакт, используй русский
- Если нужен термин, используй английский термин внутри русского текста

# 2. Будь ответственным исполнителем

- Выполнение запроса не должно приводить к ошибкам и сбоям в
  работе текущих решений и инструментов, проверь что ни чего не ломаешь,
  но если это единственный способ для успешного выполнения запроса, то
  предупреди и запроси подтверждение
- Готовь план отката изменений:
  - если изменения файлов вне системы контроля версий, то план через
    резервную копию
  - если изменения (не в файлах) совершаются через скрипт (команду), то
    план подготовить скрипт, который откатит изменения
  - если понимаешь, что не сможешь откатить изменения, то
    подготовь план для пользователя, по которому
    пользователь надёжно откатит изменения
- Если изменения нельзя будет откатить, то предупреди и запроси
- Если изменение привело к ошибке, то откати и сообщи
- После успешного применения изменений удали резервные копии и скрипты
- После успешного выполнения действий удаляй временные файлы и записи

# 3. Делай commit правильно

- Если нужно создать commit - правила в
  [commit-rules](rules/commit-rules.md)

# 4. Форматируй файлы

- Для каждого созданного или изменённого файла — определи его
  расширение и проверь наличие в [rules](lint/lint-rules.yml), и
  если оно есть, то:
  - в Win10/WSL2 вызови `pwsh ./lint/lint.ps1 -Path "<file_path>"`
  - вне Win10/WSL2 вызови `bash ./lint/lint.sh -Path "<file_path>"`
- При ошибках сообщи о них, работу продолжай

# 6 Веди журнал работы

- Если начал выполнять запрос от пользователя (или план), то запиши его
  в файл с отметкой времени, файл создай в [logs](logs), для имени файла
  используй шаблон `<agent-id>-<model-name>-<UNIX timestamp>-<uuid>.md`,
  где:
  - <твоё имя> — имя агента (cursor-assistant, assistant, и т.д.)
  - <имя модели> — конкретная модель LLM
  - `<UNIX timestamp>` — текущий UNIX timestamp
  - `<uuid>` — значение сгенерированное в формате UUID
- Если выполнил запрос от пользователя и делаешь доклад, то запиши
  доклад в тот же файл, если файла не существует, то сначала создай его
