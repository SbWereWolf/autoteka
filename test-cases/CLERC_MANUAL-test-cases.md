# Э4 — тест-кейсы для docs/foundations/CLERC_MANUAL.md

## Область

- Документ-источник: `docs/foundations/CLERC_MANUAL.md`
- Цель: проверить тестами проверяемые утверждения документа
- Формат трассировки:
  `CLERC_MANUAL.md -> утверждение -> тест-кейс -> каталог тестов`

## Тест-кейсы

### TC-CLERC-MANUAL-001

- Утверждение: основные источники данных:
  `city-list.json`, `category-list.json`, `feature-list.json`,
  `shops.json`, `frontend/public/generated/*`.
- Проверка: проверить существование всех перечисленных путей.
- Ожидаемый результат: все источники данных присутствуют в репозитории.
- Тип: automated
- Каталог тестов: `system-tests`

### TC-CLERC-MANUAL-002

- Утверждение: перед публикацией из `frontend/` выполняются
  `validate:mocks`, `check:unused-assets`, `check:data`.
- Проверка:
  1. Проверить наличие скриптов в `frontend/package.json`.
  2. Выполнить команды в корректном окружении.
  3. Смоделировать ошибку данных и проверить fail-поведение.
- Ожидаемый результат:
  - команды доступны и исполняются;
  - при ошибках данные не проходят контроль.
- Тип: automated
- Каталог тестов: `frontend/tests`

### TC-CLERC-MANUAL-003

- Утверждение: публикация выполняется командой
  `autoteka:data:import` в контейнере `php` с scope
  `city|category|feature|shop` и режимами
  `dry-run|refresh|append`.
- Проверка:
  1. Проверить сигнатуру artisan-команды.
  2. Выполнить прогоны по всем scope и режимам на тестовом наборе.
- Ожидаемый результат:
  команда поддерживает все заявленные scope/режимы.
- Тип: automated
- Каталог тестов: `backend/tests` + `deploy/tests`

### TC-CLERC-MANUAL-004

- Утверждение: для `scope=shop` обязателен `--generated-root`.
- Проверка:
  1. Запустить импорт `shop` без `--generated-root`.
  2. Запустить импорт `shop` с валидным `--generated-root`.
- Ожидаемый результат:
  - без параметра команда завершаетcя ошибкой валидации;
  - с параметром команда выполняется.
- Тип: automated
- Каталог тестов: `backend/tests`

### TC-CLERC-MANUAL-005

- Утверждение: `dry-run` не сохраняет изменения в БД.
- Проверка:
  1. Зафиксировать исходное состояние таблиц выбранного scope.
  2. Выполнить `autoteka:data:import --mode=dry-run`.
  3. Сравнить состояние БД до и после.
- Ожидаемый результат: состояние БД не изменяется.
- Тип: automated
- Каталог тестов: `backend/tests`

### TC-CLERC-MANUAL-006

- Утверждение: `refresh` очищает и перезаписывает scope,
  `append` только добавляет данные.
- Проверка:
  1. Подготовить тестовый набор и исходную БД.
  2. Выполнить импорт в режиме `refresh`.
  3. Выполнить импорт в режиме `append`.
  4. Сравнить результат с ожидаемой моделью изменений.
- Ожидаемый результат:
  режимы работают в соответствии с документацией.
- Тип: automated
- Каталог тестов: `backend/tests`

### TC-CLERC-MANUAL-007

- Утверждение: frontend-цикл подготовки включает
  `images:regen`, `images:moonshine`,
  `materialize:shop-media`, `check:data`.
- Проверка:
  1. Выполнить полный цикл команд из `frontend/`.
  2. Проверить артефакты генерации и валидность данных.
- Ожидаемый результат:
  цикл подготовки выполняется и формирует корректные данные.
- Тип: automated
- Каталог тестов: `frontend/tests`

### TC-CLERC-MANUAL-008

- Утверждение: `images:moonshine` переводит ссылки
  `generated/*.svg` -> `generated/*.png` в `shops.json`.
- Проверка:
  1. Подготовить моки с SVG-ссылками.
  2. Запустить `npm run images:moonshine`.
  3. Проверить обновление ссылок на PNG.
- Ожидаемый результат: ссылки в `shops.json` обновлены на PNG.
- Тип: automated
- Каталог тестов: `frontend/tests`

### TC-CLERC-MANUAL-009

- Утверждение: `materialize:shop-media` детерминированно проставляет
  `thumbUrl/galleryImages` в `shops.json`.
- Проверка:
  1. Дважды запустить materialize на одинаковом входе.
  2. Сравнить выходные файлы.
- Ожидаемый результат: результаты идентичны между запусками.
- Тип: automated
- Каталог тестов: `frontend/tests`

### TC-CLERC-MANUAL-010

- Утверждение: `sync:backend-media` синхронизирует ассеты в backend storage.
- Проверка:
  1. Запустить `npm run sync:backend-media`.
  2. Проверить копирование файлов в `backend/storage/app/public/generated`.
- Ожидаемый результат: backend storage содержит актуальные generated-файлы.
- Тип: automated
- Каталог тестов: `system-tests`

### TC-CLERC-MANUAL-011

- Утверждение: команда
  `autoteka:media:update-generated-paths-to-png`
  поддерживает `--dry-run`, переводит пути SVG -> PNG
  и подготавливает preview-копии.
- Проверка:
  1. Выполнить `--dry-run` и проверить отсутствие изменений в БД.
  2. Выполнить боевой запуск и проверить изменения путей/preview.
- Ожидаемый результат:
  dry-run не изменяет данные, основной режим применяет изменения.
- Тип: automated
- Каталог тестов: `backend/tests`

### TC-CLERC-MANUAL-012

- Утверждение: после публикации данные доступны через API `/api/v1/*-list`,
  карточка магазина открывается по `/shop/:code`,
  изображения доступны по `/storage/*`,
  preview в MoonShine использует актуальные PNG.
- Проверка: end-to-end проверка после импорта данных.
- Ожидаемый результат:
  API/UI/media/preview соответствуют критериям качества публикации.
- Тип: automated
- Каталог тестов: `system-tests`

### TC-CLERC-MANUAL-013

- Утверждение: при эскалации оператор передаёт `code` магазина/scope
  и вывод команды импорта.
- Проверка:
  1. Смоделировать ошибочный импорт.
  2. Проверить, что в отчёте присутствуют `code|scope` и лог команды.
- Ожидаемый результат:
  инцидентный отчёт содержит обязательные данные для эскалации.
- Тип: automated
- Каталог тестов: `system-tests`

### TC-CLERC-MANUAL-014

- Утверждение: в разделе "Смежные документы" ссылки на
  `ADMIN_MANUAL.md`, `IMPLEMENTATION.md`, `deploy/DEPLOY.md` валидны.
- Проверка: проверить существование файлов по указанным путям.
- Ожидаемый результат: все ссылки ведут на существующие документы.
- Тип: automated
- Каталог тестов: `system-tests`

## Условно проверяемые утверждения

- Описание зоны ответственности оператора (clerc) проверяется
  process-check, а не runtime-тестами.
- Дата актуализации (`2026-03-07`) подтверждается только
  процессной ревизией артефактов Э3/Э4.
