# Э4 — тест-кейсы для docs/foundations/USER_MANUAL.md

## Область

- Документ-источник: `docs/foundations/USER_MANUAL.md`
- Цель: проверить тестами проверяемые утверждения документа
- Формат трассировки:
  `USER_MANUAL.md -> утверждение -> тест-кейс -> каталог тестов`

## Тест-кейсы

### TC-USER-MANUAL-001

- Утверждение: front office позволяет выбрать город, настроить приоритет
  категорий/фишки, просматривать каталог, открывать карточку, смотреть
  галерею/контакты/режим работы/сайт и переключать тему.
- Проверка: end-to-end сценарий, покрывающий все перечисленные действия.
- Ожидаемый результат:
  каждый пользовательский сценарий выполняется из интерфейса.
- Тип: automated
- Каталог тестов: `system-tests`

### TC-USER-MANUAL-002

- Утверждение: главная страница `/` показывает каталог выбранного города.
- Проверка:
  1. Открыть `/` и выбрать город A.
  2. Проверить, что отображаются только магазины города A.
  3. Переключить на город B и повторить проверку.
- Ожидаемый результат:
  состав каталога зависит от выбранного города.
- Тип: automated
- Каталог тестов: `frontend/tests`

### TC-USER-MANUAL-003

- Утверждение: категории и фишка влияют на порядок плиток.
- Проверка:
  1. Зафиксировать порядок плиток без фильтров.
  2. Применить категории и/или фишку.
  3. Сравнить порядок до и после.
- Ожидаемый результат:
  порядок плиток меняется в соответствии с выбранными фильтрами.
- Тип: automated
- Каталог тестов: `frontend/tests`

### TC-USER-MANUAL-004

- Утверждение: выбранные город/категории/фишка сохраняются в браузере
  и восстанавливаются после перезагрузки.
- Проверка:
  1. Выбрать значения в UI.
  2. Обновить страницу.
  3. Проверить восстановление выбора.
  4. Проверить значения ключей в localStorage.
- Ожидаемый результат:
  состояние фильтров сохраняется и восстанавливается.
- Тип: automated
- Каталог тестов: `frontend/tests`

### TC-USER-MANUAL-005

- Утверждение: `/shop/:code` показывает детальную карточку магазина
  (название, режим, описание, контакты, галерея, переход на сайт).
- Проверка:
  1. Открыть валидный `/shop/<code>`.
  2. Проверить наличие всех перечисленных блоков.
- Ожидаемый результат:
  карточка содержит заявленные данные и действия.
- Тип: automated
- Каталог тестов: `frontend/tests`

### TC-USER-MANUAL-006

- Утверждение: галерея листается кнопками и свайпом.
- Проверка:
  1. На карточке с галереей проверить навигацию кнопками.
  2. На touch-эмуляции проверить свайп.
- Ожидаемый результат:
  оба способа навигации по галерее работают.
- Тип: automated
- Каталог тестов: `frontend/tests`

### TC-USER-MANUAL-007

- Утверждение: переключение темы в верхней панели применяется сразу,
  сохраняется и восстанавливается после перезагрузки.
- Проверка:
  1. Переключить тему в UI.
  2. Проверить немедленное применение.
  3. Перезагрузить страницу и проверить восстановление.
- Ожидаемый результат:
  тема применяется runtime и сохраняется в localStorage.
- Тип: automated
- Каталог тестов: `frontend/tests`

### TC-USER-MANUAL-008

- Утверждение: меню закрывается по кнопке, по overlay и по `Escape`.
- Проверка:
  1. Открыть меню и закрыть по кнопке.
  2. Открыть снова и закрыть кликом по overlay.
  3. Открыть снова и закрыть клавишей `Escape`.
- Ожидаемый результат: все три способа закрытия работают.
- Тип: automated
- Каталог тестов: `frontend/tests`

### TC-USER-MANUAL-009

- Утверждение: после закрытия меню фокус возвращается к элементу,
  из которого меню было открыто.
- Проверка:
  1. Открыть меню из `TopBar`.
  2. Закрыть меню.
  3. Проверить текущий focused element.
- Ожидаемый результат: фокус возвращается на исходный trigger.
- Тип: automated
- Каталог тестов: `frontend/tests`

### TC-USER-MANUAL-010

- Утверждение: сброс локального состояния через удаление ключей
  `autoteka_city`, `autoteka_categories`, `autoteka_feature`,
  `autoteka_theme` очищает пользовательские настройки.
- Проверка:
  1. Установить значения фильтров/темы.
  2. Выполнить скрипт удаления ключей.
  3. Перезагрузить страницу.
  4. Проверить, что состояние сброшено.
- Ожидаемый результат:
  после удаления ключей UI возвращается к состоянию по умолчанию.
- Тип: automated
- Каталог тестов: `frontend/tests`

### TC-USER-MANUAL-011

- Утверждение: при несуществующем `shop/:code` пользователь может
  вернуться в каталог и повторно открыть карточку.
- Проверка:
  1. Открыть несуществующий `/shop/<invalid-code>`.
  2. Проверить корректный fallback/ошибку.
  3. Вернуться в каталог и открыть валидный shop.
- Ожидаемый результат:
  сценарий восстановления работоспособен для пользователя.
- Тип: automated
- Каталог тестов: `frontend/tests`

### TC-USER-MANUAL-012

- Утверждение: ссылки на `ADMIN_MANUAL` и `DEPLOY` в конце документа
  валидны.
- Проверка: проверить существование файлов
  `docs/foundations/ADMIN_MANUAL.md` и `deploy/DEPLOY.md`.
- Ожидаемый результат: обе ссылки указывают на существующие документы.
- Тип: automated
- Каталог тестов: `system-tests`
