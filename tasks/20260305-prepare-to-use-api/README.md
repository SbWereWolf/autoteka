# docs/

Содержимое:

- `migration-plan-api-v1.md` — поэтапный план миграции фронта на API-контракт v1 (без сети), включая test cases и требования к проверке моков.
- `migration-checklist-api-v1.md` — чеклист для PR (галочки по этапам).

Рекомендуемый порядок работы:
1) Stage 0 (страховочные тесты)
2) Stage 1a (обогащение моков + валидатор)
3) Stage 1b (переключение на новые поля без fallback)
4) Stage 2 (справочники + state по ID)
5) Stage 3 (вывод категорий/фич на карточке)
6) Stage 4 (контакты через acceptable-contact-types)
7) Stage 5 (ApiClient без сети)
8) Stage 6 (ошибки/устойчивость)
