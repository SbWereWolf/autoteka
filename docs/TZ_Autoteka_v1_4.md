# Техническое задание (ТЗ)
## Макет клиентского приложения «Автотека» — v1.4

Дата: 03.03.2026

---

## 1. Цель
Сделать **клиентский веб‑макет** (SPA) каталога магазинов автозапчастей для демонстрации бизнес‑заказчику.

- Без бэкенда: **все данные замоканы/захардкожены**.
- Метрики не собираем, пользовательские тесты пока не проводим.

---

## 2. Область работ (scope)
Входит:
- Экран **Каталог магазинов** (плитки).
- Экран **Карточка магазина**.
- **Меню (гамбургер)**: выбор города, выбор категорий (мультивыбор), выбор фишки (одиночный выбор).
- **Сортировка** по категориям/фишке (строго по алгоритму из раздела 8).
- Переключение **6 тем** (3 стиля × 2 палитры) + сохранение выбора.
- Переход на сайт магазина:
  - кнопкой,
  - и через **«доскролл вниз»** (как «следующий чат» в Telegram) с индикатором и вибро‑откликом, где доступно.

Не входит:
- Админка/CRUD, загрузка изображений, API.

---

## 3. Технологии и ограничения
- **Vue 3.5+** (Composition API, `<script setup>`).
- **Tailwind CSS 4.1+**.
- Цвета и палитры: **только `oklch(...)`** и производные через **`color-mix(in oklch, ...)`**.
- Все мок‑данные хранятся в репозитории в **pretty format** (многострочный JSON/TS, удобно редактировать руками).

---

## 4. Брейкпоинты и адаптив (Tailwind screens)
Используем фиксированные брейкпоинты:
- `xs = 20rem` (320px)
- `sm = 24rem` (384px)
- `3xl = 48rem` (768px)
- `7xl = 80rem` (1280px)

Приоритет: **xs и sm** (самые маленькие и средние смартфоны). Дополнительно: **3xl** (планшет) и **7xl** (десктоп).

---

## 5. Данные (моки)
### 5.1. Справочники
Категории:
- Отечественные запчасти
- Китайски запчасти
- Европейские запчасти
- Японские запчасти
- Корейские запчасти

Фишки:
- самая быстрая доставка
- акции
- круглосуточно

Города:
- Барнаул (по умолчанию)
- Нижний Новгород
- Горно‑Алтайск

### 5.2. Распределение магазинов по городам
- Барнаул: **17 магазинов**
- Нижний Новгород: **3 магазина**
- Горно‑Алтайск: **9 магазинов**

### 5.3. Модель магазина
Минимальная структура:
- `id: string`
- `name: string`
- `city: string`
- `categories: string[]`
- `features: string[]`
- `workHours: string` (с переносами строк)
- `description: string`
- `contacts: { type: 'phone'|'address'|'text'|'telegram'|'whatsapp'|'email', value: string }[]`
- `siteUrl: string`
- `gallery: { kind: 'placeholder', label: string }[]` (без реальных фото)

## 6. Темы, стили и фон
### 6.1. Темы
Всего **6 тем**: **3 стиля × 2 палитры**.

Стили:
- **A** — «Чистый каталог + чипсы»
- **B** — «Тёмный + стекло»
- **C** — «Двухпанельный (по настроению макета)»

Палитры в каждом стиле:
- **Neutral**
- **Accent**

### 6.2. Переключение тем
- Переключатель (6 иконок) расположен в **sticky top bar**.
- Текущая активная тема **явно выделяется**.
- Выбор темы сохраняется в **localStorage**.

### 6.3. Фон приложения
- Для каждого стиля используется **свой тайловый фон** (как фон чатов WhatsApp).
- Фон **не затемняется** и показывается «как есть».
- При скролле фон **остаётся на месте** (`background-attachment: fixed`).

Важно:
- **Фон не должен просвечивать** через плитки, карточки, абзацы и любые текстовые панели.
- Все текстовые блоки размещаются на **непрозрачных поверхностях**.

---

## 7. Общие правила UI/UX
### 7.1. Непрозрачные поверхности
- Любой текст/контент на пёстром фоне должен иметь собственную непрозрачную подложку (`surface` / `surface-strong`).

### 7.2. Hover/tap‑реакции и «bounce»
- **Все кликабельные элементы** должны реагировать на наведение/нажатие:
  - В светлых темах элемент **слегка темнеет**.
  - В тёмных темах элемент **слегка подсветляется**.
- Для кнопок используется **тот же bounce‑эффект, что и для плиток**.
- Bounce усилен (примерно ×2 по подъёму/scale относительно базового).
- На устройствах без hover эффект должен быть заметен на **tap** (active state).

### 7.3. Анимации
- Анимации используются для: открытия/закрытия меню, dropdown, смены темы, интерактивных состояний.
- Длительности: «как сейчас реализовано», без необходимости стандартизировать до миллисекунды.

---

## 8. Сортировка (ключевая логика)
Категории и фишка **не фильтруют** список (не скрывают магазины), а **только меняют порядок**.

### 8.1. Выбор
- Фишка: выбирается **только одна** (одиночный выбор).
- Категории: выбирается **несколько** (мультивыбор).

### 8.2. Алгоритм
Пусть:
- `selectedCategories` — выбранные категории (0..N)
- `selectedFeature` — выбранная фишка (0..1)

Для магазина:
- `hasCategory = true`, если у магазина есть **хотя бы одна** из выбранных категорий
- `hasFeature = true`, если у магазина есть выбранная фишка

Шаги:
1) Делим список на 2 массива по категориям:
- `A`: `hasCategory = true` (в начало)
- `B`: `hasCategory = false` (в конец)

2) Внутри `A` и `B` делим по фишке:
- `A1`: есть категория и есть фишка
- `A2`: есть категория и нет фишки
- `B1`: нет категорий и есть фишка
- `B2`: нет категорий и нет фишки

3) Склеиваем в порядке:
- a) `A1`
- b) `A2`
- c) `B1`
- d) `B2`

Порядок внутри `A1/A2/B1/B2` не важен.

## 9. Экран 1 — Каталог магазинов
### 9.1. Сетка плиток
- `xs`/`sm`: **2 плитки в ряд**
  - `xs`: gap минимальный (плотно)
  - `sm`: gap больше
- `3xl`: **3 плитки в ряд**
- `7xl`: **4 плитки в ряд**
- На больших экранах контейнер ограничивается по ширине (плитки не растягиваются бесконечно).

### 9.2. Плитка магазина
- Без картинок, фон‑градиент (монотонный) + **название магазина**.
- Название:
  - отдельный display‑шрифт по стилю A/B/C,
  - текст с обводкой (stroke), чтобы читался на фоне.

### 9.3. Навигация
- Тап по плитке → экран карточки магазина.

---

## 10. Экран 2 — Меню (гамбургер)
### 10.1. Открытие/закрытие
- Открытие: кнопка «≡» в top bar.
- Закрытие: крестик **или клик/тап мимо панели** (по затемнению).

### 10.2. Содержимое меню
1) **Город (фильтр)**
- Dropdown/select.
- Всегда выбран один город.
- При смене города меняется набор плиток (17/3/9).

2) **Категории (сортировка, мультивыбор)**
- Несколько категорий могут быть выбраны одновременно.
- Влияют только на порядок (см. сортировку).

3) **Фишка (сортировка, одиночный выбор)**
- Dropdown/select.
- По умолчанию выбрано значение: **«самая быстрая доставка»**.

---

## 11. Экран 3 — Карточка магазина
### 11.1. Состав блоков
- Название магазина (на непрозрачной подложке)
- Галерея (заглушки), свайп влево/вправо
- Оверлей «режим работы»
- Описание
- Контакты (кликабельные)
- Кнопка «Перейти на сайт»

### 11.2. Адаптив карточки магазина
- На смартфонах (xs/sm) текущая высота галереи — как есть.
- На планшете/десктопе (3xl/7xl):
  - горизонтальные поля (gutter) больше,
  - галерея по высоте **примерно в 1.5 раза ниже** (за счёт aspect‑ratio),
  - контентный контейнер ограничен по ширине.

### 11.3. Контакты
Кликабельность:
- phone → `tel:`
- email → `mailto:`
- telegram/whatsapp → ссылки
- address → текст (опционально ссылка на карты)

### 11.4. Переход на сайт через «доскролл вниз»
- При достижении низа страницы появляется индикатор.
- При «перетягивании» до порога текст меняется на **«Отпустите для перехода»**.
- Должно работать:
  - на touch (жест),
  - на desktop/trackpad через wheel (аналог «отпустили» = краткая пауза).
- Вибро‑отклик: `navigator.vibrate(...)`, если поддерживается.
- Открытие сайта: **в этой вкладке**.

## 12. Use cases (реализованные сценарии)
1) **Просмотр каталога**
- Открыть приложение → отображается каталог Барнаул (17 плиток).

2) **Выбор города**
- Открыть меню → выбрать город → каталог показывает магазины выбранного города.

3) **Сортировка по категориям и фишке**
- В меню выбрать 0..N категорий и 0..1 фишку → список переупорядочивается по алгоритму a/b/c/d.

4) **Просмотр карточки магазина**
- В каталоге нажать плитку → открыть карточку.

5) **Просмотр галереи**
- На карточке магазина свайпать галерею влево/вправо.

6) **Переход на сайт магазина**
- Нажать кнопку «Перейти на сайт» **или** выполнить «доскролл вниз» до порога и отпустить.

7) **Смена темы**
- В top bar выбрать тему → меняется стиль+палитра, выбор сохраняется.

## 13. Критерии приёмки (acceptance)
- Данные замоканы и редактируются в pretty JSON.
- Города и распределение 17/3/9 работают.
- Меню закрывается по крестику и по клику/тапу мимо.
- Сортировка соответствует алгоритму a/b/c/d.
- Все кликабельные элементы имеют hover/tap‑реакцию и bounce.
- 6 тем переключаются, активная тема видна, выбор сохраняется.
- Фон:
  - есть всегда,
  - разный для стилей A/B/C,
  - fixed при скролле,
  - нигде не просвечивает через контент.
- Карточка магазина:
  - адаптив по брейкпоинтам,
  - галерея снижает высоту на 3xl/7xl,
  - контакты кликабельны,
  - «доскролл вниз» работает на touch и wheel.
