# ТЗ v1.3 — «Автотека» (клиентский макет, данные замоканы)

## 1) Цель
Сделать **клиентское веб-приложение-макет** каталога магазинов (без бэкенда), чтобы **показать бизнес-заказчику** UX и дизайн-концепт.

Метрики не собираем, пользовательских тестов пока нет.

Основание: исходный документ (структура, сценарии, меню, “перескролл вниз”, mobile-first и т.д.).

---

## 2) Технологии и ограничения
- Только клиент (SPA), все данные — моки/хардкод.
- **Vue 3.5+ (Composition API, `<script setup>`).**
- **Tailwind CSS 4.1+**.
- Цветовая палитра/переменные темы: **только `oklch(...)`** (включая альфу через `oklch(... / a)`).
- Открытие сайта магазина: **в этой вкладке**.
- Анимации: **везде на все эффекты**, с разными длительностями (см. раздел 12).

---

## 3) Моки и формат хранения данных
### 3.1. Формат
- Данные хранить в проекте как JSON/TS-объекты в **pretty format** (многострочно, читабельно, не minify).

### 3.2. Справочники (из исходника)
**Категории:**
- Отечественные запчасти
- Китайски запчасти
- Европейские запчасти
- Японские запчасти
- Корейские запчасти

**Фишки магазинов:**
- самая быстрая доставка
- акции
- круглосуточно

**Города (исправлена опечатка):**
- Барнаул
- Нижний Новгород
- Горно-Алтайск

### 3.3. Распределение магазинов по городам
- Барнаул (по умолчанию): **17 магазинов**
- Нижний Новгород: **3 магазина**
- Горно-Алтайск: **9 магазинов**

---

## 4) Модель магазина (минимум)
- `id: string`
- `name: string`
- `city: string` (одно из значений справочника)
- `categories: string[]`
- `features: string[]`
- `workHours: string` (с переносами строк)
- `description: string`
- `contacts: { type: 'phone'|'address'|'text'|'telegram'|'whatsapp'|'email', value: string }[]`
- `siteUrl: string`
- `gallery: { kind: 'placeholder', label: string }[]` (заглушки, без реальных изображений; свайп)

---

## 5) Экраны и навигация
### 5.1. Стартовая страница (каталог плиток)
- Стартовая: список магазинов плитками.
- Город по умолчанию: **Барнаул**, отображается **17 плиток**.
- Плитки **без картинок**: фон (цвет/градиент/паттерн) + **название магазина**.

**Название магазина на плитке:**
- “Особый” шрифт зависит от выбранного стиля A/B/C.
- Текст с **обводкой** для читаемости на фоне.

> Бейджи/мини-метки на плитках — **не делать**, оставить как “Future”.

**Переход:** тап по плитке → страница магазина.

### 5.2. Страница магазина
Состав:
- Название
- Галерея (свайп), заглушки 3:2
- Оверлей с режимом работы
- Описание
- Контакты (кликабельные)
- Кнопка/ссылка “Перейти на сайт”
- “Доскролл вниз → переход на сайт” (см. 6)

---

## 6) “Доскролл вниз → переход на сайт”
Поведение как “следующий чат” в Telegram.
- В конце страницы при протягивании вниз:
  - показываем **визуальный индикатор прогресса**,
  - при достижении порога показываем текст **«Отпустите для перехода»**,
  - при отпускании после порога → открыть `siteUrl` **в этой вкладке**.
- “Haptic”: если устройство поддерживает Web Vibration API — короткая вибрация; если нет — только визуальный индикатор.

---

## 7) Sticky top bar
- Sticky на всех экранах.
- Содержит:
  1) “≡” (гамбургер)
  2) Название “Автотека”
  3) Переключатель темы (6 иконок)

> Выбор города находится **внутри меню** (см. 8).

---

## 8) Меню (гамбургер)
Открытие/закрытие — стандартное (крестик/тап по оверлею/назад).

Внутри меню:

### 8.1. Город (фильтр)
- Dropdown/select, один город всегда выбран и обозначен.
- Смена города меняет набор плиток согласно распределению 17/3/9.

### 8.2. Категории (мультивыбор)
- Список кнопками, можно выбрать **несколько**.
- Это **не фильтр**, а сортировка (см. 9).

### 8.3. Фишка (одиночный выбор)
- В неактивном состоянии показывает выбранное значение.
- По нажатию открывается список **в одну колонку**.
- Клик по элементу: выбирает, закрывает.
- Клик мимо: закрывает без изменения.
- Значение по умолчанию: **«самая быстрая доставка»**.

---

## 9) Сортировка (строго фиксированный алгоритм)
Применяется к списку магазинов **после фильтра по городу**.

Ограничения:
- Фишка: 0..1 выбранная (по умолчанию выбрана).
- Категории: 0..N выбранных.

Определения:
- `hasCategory` = у магазина есть хотя бы одна категория из `selectedCategories`
- `hasFeature` = у магазина есть выбранная фишка

Алгоритм:
1) Разбить список на:
- `A` (hasCategory = true) → в начало
- `B` (hasCategory = false) → в конец

2) Внутри `A` и `B`:
- `A1` (есть фишка), `A2` (нет фишки)
- `B1` (есть фишка), `B2` (нет фишки)

3) Склеить:
a) `A1`  
b) `A2`  
c) `B1`  
d) `B2`

Порядок внутри A1/A2/B1/B2 не важен.

---

## 10) Контакты (кликабельные)
На странице магазина:
- phone → `tel:`
- email → `mailto:`
- telegram/whatsapp → ссылки
- address → кликабельность опционально (можно ссылку на карты), минимум — копируемый текст

---

## 11) Темы оформления: 6 тем = 3 стиля × 2 палитры
### 11.1. Стили
- **A**: “Чистый каталог + чипсы”
- **B**: “Тёмный + стекло”
- **C**: “Двухпанельный (как на скрине)” — как стиль (десктоп-layout всё равно сетка)

### 11.2. Палитры
Для каждого стиля:
- **Palette 1: Neutral**
- **Palette 2: Accent**

### 11.3. Что меняет тема (стиль+палитра)
- цвета (фон/поверхности/текст/акценты/состояния),
- радиусы, тени/blur, плотность интерфейса,
- фон-паттерн (Telegram/WhatsApp vibe),
- Фон приложения: **орнамент/узор как фон чата WhatsApp** (градиентами, без картинок), подстраивается под текущую тему.
- Фон плиток: **монотонный, но более насыщенный** (выраженный tint от accent), без реальных изображений.

- шрифтовой акцент (названия плиток по стилям)

Переключение:
- 6 иконок в top bar, тема применяется с анимацией.
- Сохранение выбора: localStorage.

---

## 12) Адаптив и брейкпоинты Tailwind (фикс)
Брейкпоинты (синхронизировать в Tailwind):
- `xs` = 20rem (320px)
- `sm` = 24rem (384px)
- `3xl` = 48rem (768px)
- `7xl` = 80rem (1280px)

Приоритет: `xs` и `sm`, дополнительно `3xl` и `7xl`.

Десктоп: **обычная сетка 3–4 колонки**.

---

## 13) Анимации (везде, разные длительности)
Анимировать:
- меню, dropdown’ы (город/фишка), кнопки/чипсы,
- смену темы,
- индикатор “доскролл вниз”.

Рекомендуемые диапазоны:
- Tap/active: 80–120ms
- Dropdown: 140–200ms
- Меню: 200–280ms
- Переходы блоков: 180–260ms
- Смена темы: 240–420ms

---

## 14) Acceptance criteria
Готово, если:
1) Барнаул показывает 17 плиток; Нижний Новгород — 3; Горно-Алтайск — 9.
2) Меню: город (фильтр), категории (мультивыбор), фишка (одиночная) работают.
3) Сортировка соответствует схеме a/b/c/d.
4) Плитки без картинок, названия с обводкой, шрифты/стиль меняются при теме.
5) 6 тем (3×2) переключаются из top bar и сохраняются.
6) Страница магазина содержит все блоки, контакты кликабельны.
7) “Доскролл вниз” показывает индикатор + «Отпустите для перехода» + открывает сайт в этой вкладке, вибрация где поддерживается.
