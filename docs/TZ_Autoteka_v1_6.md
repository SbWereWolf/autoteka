# Техническое задание (ТЗ)
## Макет клиентского приложения «Автотека» — v1.6

Дата: 2026-03-02

---

## 1. Цель
Сделать **клиентский веб‑макет (SPA)** каталога магазинов автозапчастей для демонстрации бизнес‑заказчику.

- Без бэкенда: **все данные замоканы/захардкожены**.
- Метрики не собираем, пользовательские тесты пока не проводим.

---

## 2. Scope (что делаем)
Входит:
- Экран **Каталог магазинов** (плитки).
- Экран **Карточка магазина**.
- **Меню (гамбургер)**: выбор города, выбор категорий (мультивыбор), выбор фишки (одиночный выбор).
- **Сортировка** по категориям/фишке (строго по алгоритму).
- Переключение **6 тем** (3 стиля × 2 палитры) + сохранение выбора.
- Переход на сайт магазина:
  - кнопкой,
  - и через **«доскролл вниз»** с индикатором и вибро‑откликом (где доступно).

Не входит:
- Админка/CRUD, API, загрузка пользовательских изображений.

---

## 3. Технологии и требования к коду
- **Vue 3.5+** (Composition API, `<script setup>`).
- **Tailwind CSS 4.1+**.
- Цвета/палитры: **только `oklch(...)`** и **`color-mix(in oklch, ...)`**.
- Мок‑данные: **pretty format** (многострочный JSON/TS), удобно редактировать руками.

---

## 4. Брейкпоинты (Tailwind screens) и адаптив
Фиксированные брейкпоинты:
- `xs = 20rem` (320px)
- `sm = 24rem` (384px)
- `3xl = 48rem` (768px)
- `7xl = 80rem` (1280px)

Приоритет: **xs и sm** (самые маленькие и средние смартфоны). Дополнительно: **3xl** (планшет), **7xl** (десктоп).

## 5. Данные (моки)
### 5.1. Справочники
Категории:
- Отечественные запчасти
- Китайски запчасти
- Европейские запчасти
- Японские запчасти
- Корейские запчасти

Фишки:
- самая быстрая доставка *(по умолчанию выбрана)*
- акции
- круглосуточно

Города:
- Барнаул *(по умолчанию)*
- Нижний Новгород
- Горно‑Алтайск

### 5.2. Распределение магазинов по городам
- Барнаул: **17**
- Нижний Новгород: **3**
- Горно‑Алтайск: **9**

### 5.3. Модель магазина (минимум)
- `id: string`
- `name: string`
- `city: string`
- `categories: string[]`
- `features: string[]`
- `workHours: string` (с переносами строк)
- `description: string`
- `contacts: { type: 'phone'|'address'|'text'|'telegram'|'whatsapp'|'email', value: string }[]`
- `siteUrl: string`
- `thumbUrl?: string` — миниатюра (для части магазинов)
- `galleryImages?: string[]` — изображения галереи (для части магазинов)
- `gallery?: { kind: 'placeholder', label: string }[]` — заглушки, если нет `galleryImages`

### 5.4. Генерация изображений для моков
Для **примерно половины магазинов** используются **сгенерированные изображения**:
- Соотношения сторон: **1:1**, **2:3**, **3:2**
- Размеры: диапазон **0.5× … 1.5×** от «стандартного», шаг **0.25×**
  - множители: `0.5, 0.75, 1.0, 1.25, 1.5`
- Хранение: `public/generated/…`
- Использование: пути `/generated/<file>.png` в мок‑данных.
- Отображение:
  - В каталоге миниатюра **опционально** показывается внутри плитки.
  - В карточке магазина галерея показывает `galleryImages`, иначе — заглушки.

## 6. Темы/стили и фон
### 6.1. 6 тем (3 стиля × 2 палитры)
Стили:
- **A** — «Чистый каталог + чипсы»
- **B** — «Тёмный + стекло»
- **C** — «Двухпанельный (по настроению)»

Палитры в каждом стиле:
- **Neutral**
- **Accent**

### 6.2. Переключение тем
- Переключатель (6 кнопок/иконок) расположен в **sticky top bar**.
- Активная тема **явно выделяется**.
- Выбор темы сохраняется в **localStorage**.

### 6.3. Фон
- Для каждого стиля используется **свой тайловый фон** (как WhatsApp chat background).
- Фон **обязателен всегда** (если фона нет — дефект/нет приёмки).
- Фон **не затемняется**, отображается «как есть».
- При скролле фон **фиксирован** (`background-attachment: fixed`).
- Фон **не должен просвечивать** через плитки/карточки/абзацы/панели: все контент‑поверхности непрозрачные.

---

## 7. Общие правила UI/UX
### 7.1. Непрозрачные поверхности
Любой текст/контент поверх пёстрого фона размещается на **своей непрозрачной подложке** (`surface` / `surface-strong`).

### 7.2. Hover/tap и bounce (обязательно для всего кликабельного)
- Все элементы, на которые можно нажать, должны **предлагать клик**:
  - в **светлых** темах элемент на hover **слегка темнеет**
  - в **тёмных** темах элемент на hover **слегка подсветляется**
- Для кнопок используется **тот же bounce‑эффект, что у плиток**.
- Bounce усилен (≈×2 по подъёму/scale).
- На устройствах без hover эффект должен быть заметен на **tap**.
- Hover должен работать на всех брейкпоинтах, включая случаи, когда эмуляция ограничивает CSS `:hover`.

### 7.3. Анимации
Анимации используются для: меню, dropdown, смены темы, интерактивных состояний, индикатора доскролла.
Длительности — «как в текущей реализации» (без фиксации до мс).

---

## 8. Логика сортировки (ключевая)
Город — **фильтр** (меняет набор магазинов). Категории и фишка — **только сортировка** (не скрывают элементы).

- Фишка: выбирается **только одна**.
- Категории: выбирается **несколько**.

Алгоритм (строго):
1) Делим список на 2 массива по категориям:
- `A`: есть хотя бы одна выбранная категория → в начало
- `B`: нет выбранных категорий → в конец
2) Внутри `A` и `B` делим по фишке:
- `A1`: есть категория и есть фишка
- `A2`: есть категория и нет фишки
- `B1`: нет категорий и есть фишка
- `B2`: нет категорий и нет фишки
3) Склейка:
- a) `A1`
- b) `A2`
- c) `B1`
- d) `B2`

Порядок внутри подмассивов не важен.

## 9. Экран 1 — Каталог магазинов
### 9.1. Заголовок
Заголовок/город/количество отображаются на **непрозрачной панели**.

### 9.2. Сетка плиток (responsive)
- `xs`/`sm`: **2 плитки в ряд**
  - `xs`: gap минимальный (плотно)
  - `sm`: gap больше
- `3xl`: **3 плитки в ряд**
- `7xl`: **4 плитки в ряд**
- На больших экранах контейнер ограничивается по ширине (плитки не растягиваются бесконечно).
- На планшете/десктопе gap увеличен значительно (≈×3 относительно мобилы).

### 9.3. Плитка магазина
- Монотонный фон + внутренний орнамент (по стилю) + название магазина.
- Название на **непрозрачной плашке**, display‑шрифт зависит от стиля A/B/C.
- Текст названия с обводкой (stroke) для читаемости.
- Для части магазинов показывается миниатюра (`thumbUrl`) внутри плитки.

### 9.4. Действие
Клик по плитке → переход в карточку магазина.

---

## 10. Экран 2 — Меню (гамбургер)
### 10.1. Открытие/закрытие
- Открытие: кнопка «≡» в top bar.
- Закрытие: крестик **или клик/тап мимо панели**.

### 10.2. Секции меню
1) **Город** (dropdown, один выбран всегда) — фильтр.
2) **Категории** (мультивыбор) — сортировка.
3) **Фишка** (dropdown, одиночный выбор) — сортировка.
   - В закрытом виде показывает выбранную фишку.
   - По клику открывает список в одну колонку.
   - Клик мимо — закрывает без изменения.

---

## 11. Экран 3 — Карточка магазина
### 11.1. Верхняя зона
- Кнопка «Назад» (интерактивная, с bounce).
- Название магазина на **непрозрачной панели**.

### 11.2. Галерея
- Свайп/кнопки навигации.
- Если есть `galleryImages`, показывать изображения (`object-fit: cover`).
- Иначе показывать заглушки.
- На `xs/sm` галерея выглядит как 3:2.
- На `3xl/7xl` галерея **ниже по высоте примерно в 1.5 раза** (через иной aspect‑ratio).

### 11.3. Контент
- Режим работы (оверлей на галерее) на непрозрачной подложке.
- Описание (карточка).
- Контакты:
  - телефон → `tel:`
  - email → `mailto:`
  - telegram/whatsapp → ссылка
  - адрес → текст (опционально ссылка на карты)

### 11.4. Переход на сайт
- Кнопка «Перейти на сайт» (в этой вкладке).
- Дополнительно: **«доскролл вниз»** в конце страницы:
  - появляется индикатор прогресса,
  - при достижении порога — «Отпустите для перехода»,
  - открытие работает и для touch (жест), и для wheel/trackpad,
  - вибро‑отклик, где доступно.

---

## 12. Use cases (реализованные сценарии)
1) Открыть приложение → увидеть каталог (Барнаул, 17 шт).
2) Открыть меню → выбрать город → список меняется на 3 или 9 магазинов.
3) Открыть меню → выбрать категории (несколько) → порядок списка меняется.
4) Открыть меню → выбрать фишку (одну) → порядок списка меняется.
5) Открыть магазин из каталога → перейти в карточку.
6) Просмотреть галерею (свайп/кнопки).
7) Перейти на сайт магазина кнопкой.
8) Дойти до конца страницы и сделать «доскролл вниз» → перейти на сайт магазина.

## 13. Критерии приёмки
- Фон по стилям отображается всегда и не просвечивает через контент.
- 29 магазинов в моках (17/3/9 по городам).
- Сортировка соответствует алгоритму a/b/c/d.
- Все кликабельные элементы имеют hover/tap‑реакцию и bounce.
- 6 тем переключаются и сохраняются; активная тема выделена.
- В половине магазинов используются реальные изображения (thumb + галерея) по правилам генерации.
- Доскролл‑переход работает (touch и wheel) и показывает индикатор.
